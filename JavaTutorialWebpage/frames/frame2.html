Modyfikatory dostępu

W Javie istnieją trzy modyfikatory dostępu: private, public i protected.

Modyfikator private oznacza, że dostęp do danej zmiennej lub metody jest możliwy tylko wewnątrz klasy, w której została ona zdefiniowana.
Modyfikator public oznacza, że dostęp do danej zmiennej lub metody jest możliwy z każdego miejsca w kodzie.
Modyfikator protected oznacza, że dostęp do danej zmiennej lub metody jest możliwy wewnątrz klasy oraz we wszystkich klasach dziedziczących po tej klasie oraz w ramach tego samego pakietu. 

public class Czlowiek {
    private String imie;
    public String nazwisko;
    protected int wiek;
}

W powyższym przykładzie zmienna imie jest dostępna tylko wewnątrz klasy Czlowiek, zmienna nazwisko jest dostępna z każdego miejsca w kodzie, a zmienna wiek jest dostępna wewnątrz klasy Czlowiek oraz we wszystkich klasach dziedziczących po Czlowiek.

Każda metoda musi posiadać modyfikator dostępu.
Niektóre metody powinny posiadać modyfiaktor private, jeżeli nie powinny być dostępne w innych klasach. Najczęstszymi prywatnymi metodami są metody pomocnicze, które są przydatne tylko wewnątrz danej klasy. 

public class Obliczenia {
    private double liczba1;
    private double liczba2;
    private double suma;

    //konstruktor, który pobiera dane oraz wywołuje metodę obliczającą sume
    public Obliczenia(double liczba1, double liczba2) {
        this.liczba1 = liczba1;    
        this.liczba2 = liczba2;
        obliczSume();
    }

    //Prywatna metoda obliczająca sumę dwóch liczb
    private double obliczSume() {
        suma = liczba1 + liczba2;
    }

    //Publiczna metoda wyświetlająca sumę
    public String wyswietlSume() {
        System.out.println("Suma: "+suma);
    }
}
